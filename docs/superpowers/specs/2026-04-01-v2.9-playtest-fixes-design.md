## WS1: Protocol Resilience

### Problem

The tags feature (v0.3.0) appended new fields to HANDSHAKE, SAVE_QUEST, and writeQuest/readQuest packets. Old clients can't deserialize the new format, so sync fails silently and they see an empty quest board.

Additionally, existing production servers are missing the `quest_tags` table and `predefined-tags` config because neither is auto-created on upgrade.

### Design

#### Protocol version negotiation

- New clients send protocol version as the first field in `REQUEST_SYNC` (before existing payload). Value: `1`.
- Server stores per-player protocol version in `Map<UUID, Integer>` inside `ServerPacketHandler`. Players that send a `REQUEST_SYNC` without a leading version byte = v0 (legacy).
- Detection: server wraps `REQUEST_SYNC` parsing in a try/catch. If the first VarInt doesn't match a known version, rewind and parse as v0 (no version prefix).

#### Backward-compatible packet encoding

- `writeQuest()` accepts a protocol version parameter. For v0: omits tags. For v1+: includes tags.
- `writeHandshake()` same pattern: v0 omits `predefinedTags` only. The v0 format is: `bluemapUrl, pendingRequestCount, pinnedQuestIds, playerUuid, mapNames` (mapNames was added pre-tags and v0 clients already handle it).
- Going forward: all new fields appended at end. Readers use `isReadable()` guards for trailing-optional fields. Each packet prefixed with a 2-byte length so readers can skip unknown trailing bytes.

#### DB migration

- `DataManager.init()`: after creating tables, check if `quest_tags` exists via `SELECT name FROM sqlite_master WHERE type='table' AND name='quest_tags'`. If missing, create it. Same pattern as existing `migratePinnedQuests()`.

#### Config defaults

- `Config` constructor: if `predefined-tags` key is missing from loaded config, write defaults and save. Default seed tags: `Build`, `Expeditions`.

#### Tag list sync (new packet)

- New S2C packet type: `SYNC_TAGS` (add to `PacketType` enum).
- Payload: list of all distinct tags from `quest_tags` table + seed tags from config (deduplicated).
- Sent after handshake alongside `SYNC_MY_QUESTS` / `SYNC_SERVER_QUESTS`.
- Only sent to v1+ clients.
- Client stores in `ClientSession.serverTags` (replaces `predefinedTags`).

#### Deployment

Server-only deploy + restart. Old clients immediately get quests back (server sends v0-format packets). New clients get full tag support after mod update.

## WS3: Bug Fixes

### WS3a: Wiki-links not working

**Symptoms:** Typing `[[Bonsoir]]` in description doesn't render as a link. No autocomplete dropdown appears on `[[`.

**Investigation targets:**
- `AutocompleteDropdown.java` -- is the quest list populated after sync? Does it attach to the content field?
- `ClientCache` -- are server quests available when the dropdown queries?
- `MarkdownRenderer.preprocessWikiLinks()` -- is it being called on quest content in view mode?
- Server-side `WikiLinkResolver` -- does it have quests to resolve against?

**Approach:** Debug the data flow from sync -> cache -> autocomplete and from content -> markdown renderer -> wiki-link preprocessing. Fix where the chain breaks.

### WS3b: BlueMap "View on map" links broken

**Symptoms:** All quests link to `#world_new:0:0:0:1500:0:0:0:0:perspective` regardless of quest coordinates.

**Investigation targets:**
- Client-side URL builder that constructs the BlueMap hash fragment
- Whether quest coordinates are being read from `QuestData.coordinates()` or defaulting
- The BlueMap URL format: `#map:x:y:z:zoom:yaw:pitch:roll:perspective`

**Approach:** Trace the URL construction code, verify coords are non-null in the quest data, fix the interpolation.

## WS2: Tag UI Overhaul (design approval required)

### Colour palette

Replace 8-colour palette with 16+ colours using golden angle hue spacing (137.5 degrees around HSL wheel). Keep deterministic hash-based assignment so same tag = same colour everywhere.

### Tag chips

Design pass with `/frontend-design` + playground before implementing. Both screens need work:
- **Edit screen:** proper chips with rounded background, padding, remove button. Well-spaced and delimited.
- **Detail screen:** inline chips (not collapsed plain text with spaces).

### Tag picker

- Shows all tags in use on the server (from `SYNC_TAGS`) + config seed tags
- Clickable list with coloured chips -- tap to add
- Custom tag input still available for creating new tags
- Design pass with playground to explore layout options

### Execution

Blocked on WS1 (needs `SYNC_TAGS` packet). Design pass happens in a separate session with user approval before implementation.
