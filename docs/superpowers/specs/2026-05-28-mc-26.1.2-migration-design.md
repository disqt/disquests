# Disquests MC 26.1.2 Migration

Migrate disquests (Fabric client mod + PaperMC server plugin) from MC 1.21.11 to MC 26.1.2. Version bump 0.4.0 → 0.4.1. 26.1.2-only, no backwards compat.

## Root Cause Context

MC 26.x ships unobfuscated — Yarn mappings no longer exist. Fabric Loom 1.16 dropped `modImplementation` and the `mappings` configuration. The plugin ID changed from `fabric-loom` to `net.fabricmc.fabric-loom`. All verified locally by building the Fabric example mod with Kotlin DSL.

## Phase 2 — Build Tooling (Java 25)

### Spotless (google-java-format)
Spotless 7.0.3 bundles google-java-format ~1.24.0 which uses `DeferredDiagnosticHandler.getDiagnostics()` — return type changed from `Queue` to `List` in Java 25, causing `NoSuchMethodError`.

**Fix:** Spotless 7.0.3 → **8.5.1** (pulls google-java-format 1.28.0+ which supports Java 25). Pin google-java-format **1.34.0** explicitly for latest fixes.

### Error Prone + NullAway
Error Prone 2.49.0 may have classpath issues on Java 25 (`NoClassDefFoundError: DescendantOf`). NullAway 0.13.1 → **0.13.4** (includes JDK 27 EA fixes, JDK 25 well-supported).

**Fix:** NullAway 0.13.1 → **0.13.4**. Error Prone stays at 2.49.0 (latest). If EP still fails after artifact alignment, disable conditionally on Java 25.

### libs.versions.toml changes
| Property | Old | New |
|----------|-----|-----|
| spotless | 7.0.3 | 8.5.1 |
| nullaway | 0.13.1 | 0.13.4 |
| errorprone-core | 2.49.0 | 2.49.0 (unchanged, latest) |

### Spotless config change
In root `build.gradle.kts`, pin google-java-format explicitly:

### API Changes Beyond Imports

**Input event classes restructured:**
- Yarn `Click` (in `client.gui`) → Mojang `MouseButtonEvent` (in `client.input`) — record with `x()`, `y()`, `buttonInfo()` instead of simple click data
- Yarn `CharInput` → `CharacterEvent` — record with `codepoint()`
- Yarn `KeyInput` → `KeyEvent` — record with `key()`, `scancode()`, `modifiers()`
- Method signatures in Screen/Components that accept these types will change

**DrawContext → GuiGraphicsExtractor:**
- Method signatures may differ. Verify `fill()`, `drawText()`, `drawTooltip()` etc. still match.

**ClickEvent sealed hierarchy:**
- Yarn: `new ClickEvent.RunCommand(cmd)`, `new ClickEvent.OpenUrl(uri)` — may still work since these are inner record classes. Verify against the 26.1.2 jar.

**Text → Component:**
- `Text.literal(...)` → `Component.literal(...)`
- `Text.translatable(...)` → `Component.translatable(...)`
- `Text.empty()` → `Component.empty()`

### Mixin Targets
- `InGameHudMixin`: target class `net.minecraft.client.gui.hud.InGameHud` → `net.minecraft.client.gui.Gui`
- `InventoryBadgeMixin`: target class `net.minecraft.client.gui.screen.ingame.HandledScreen` → `net.minecraft.client.gui.screens.inventory.AbstractContainerScreen`
- Method descriptors in `@Inject` annotations must match the new method signatures (especially `mouseClicked` parameter types)

### Wildcard Import
- `net.minecraft.client.gui.*` in MultiLineTextFieldWidget — this will resolve to the new package contents automatically

## Phase 4 — Build, Fix, Verify

1. Apply all changes
2. Build `:common:compileJava` first (no MC deps, just Error Prone fix needed)
3. Build `:client:compileJava` — fix remaining compilation errors iteratively
4. Build `:server:compileJava` — should be straightforward (Paper API, no MC remapping)
5. Run unit tests (`common:test`, `server:test`)
6. Full `./gradlew build -x spotlessCheck` then with spotless

## Out of Scope
- E2E integration tests (require running server + client, separate session)
- CI pipeline changes (Java 25 runners)
- Mixin refactoring beyond what's needed for compilation
