# v2.9 Playtest Feedback — 2026-04-01

Disquests client v0.3.1 user testing session.

**Cross-cutting requirement:** Every fix must have E2E test coverage.

## Feedback

1. **No quests visible on disqt.com server** — Investigated. Two issues:
   - Duplicate plugin jar (`Disquests.jar` + `disquests-paper-0.3.0.jar`) caused "Ambiguous plugin name" error. **Fixed**: removed stale jar, deployed v0.3.1, restarted.
   - `quest_tags` table not auto-created on existing DBs (plugin only creates on fresh DB). Config also missing `predefined-tags` defaults. **Needs code fix**: migration for quest_tags + config defaults on upgrade.

2. **Tag colours too similar** — Three tags ("piwigord", "test", "build") all appear in nearly the same shade of purple/magenta. Colours feel like they're incrementally stepping through an array rather than being visually distinct. Should be random with better spread/contrast between adjacent tags.

3. **Tag UI layout/alignment needs work** — Tags aren't well aligned, positioned, or delimited on the creation/edit screen. Needs a design pass. Action: use `/frontend-design` + playground to explore options before implementing.

4. **Tag UI on quest detail screen also weird** — Shows "Tags (3) v" as a collapsible section with tags listed inline as plain text ("piwigord test build") separated only by spaces, no chips/badges/delimiters. Same colours-too-similar issue. Include this screen in the design pass from #3.

5. **Tag picker says "No predefined tags"** — Expected server-defined tags to choose from (e.g. common categories). Either predefined tags were never configured on the server, or the feature to define them is missing/incomplete. Predefined tags should be config-driven (server-side config file), so server admins can define their own set.

6. **Wiki-link (`[[quest name]]`) functionality broken** — Typing `[[Bonsoir]]` in the description doesn't render as a link. Invalid names aren't visually distinguished (e.g. no different underline style for unresolved links). No autocomplete/dropdown with suggestions when typing inside `[[ ]]`.

7. **Tag search UX too bare** — Searching/adding tags should be more user-friendly. Consider a clickable list, visual picker, or similar. Action: use `/frontend-design` + playground to explore tag picker options.

8. **BlueMap "View on map" link broken** — Quests with coordinates all link to the default view (`#world_new:0:0:0:1500:0:0:0:0:perspective`) instead of zooming to the quest's actual coordinates.

9. **Tag picker should show existing tags from the server** — Instead of a static predefined list, the picker should show all tags currently in use across server quests (crowdsourced/organic). Plus a small set of config-driven "seed" tags that always appear (e.g. Build, Expeditions). Replaces the old predefined-tags-only approach.

10. **Old clients see empty quest board (protocol breaking change)** — Root cause: the tags feature appended new fields to HANDSHAKE, SAVE_QUEST, and writeQuest/readQuest packets. Old clients can't deserialize the new format, so sync fails silently. Fix: add protocol versioning (client announces version), trailing-optional fields with `isReadable()` guards, and length-prefixed packets so old readers can skip unknown bytes. Open/closed principle -- extend without breaking existing clients.
